## <h1 className="bg-gradient-to-r from-blue-500 to-purple-500 inline-block text-transparent bg-clip-text text-5xl">Git কি ?</h1>

গিট হলো একটি `ভার্সন কন্ট্রোল` টুল, যা আপনার কোড এবং ফাইলের বিভিন্ন সংস্করণ ট্র্যাক করতে সাহায্য করে। এটি বিশেষ করে সফটওয়্যার ডেভেলপমেন্টে ব্যবহৃত হয়, কিন্তু যেকোনো ধরনের ফাইলের পরিবর্তন ট্র্যাক করার জন্য এটি কার্যকর।

# ভার্সন কন্ট্রোল কি ?

---

ধরুন, আপনি আপনার কম্পিউটারে একটি ডকুমেন্ট লিখছেন। আজ আপনি কিছু লিখছেন, এবং কালও হয়তো কিছু যোগ করবেন। কিন্তু একদিন, আপনি উপলব্ধি করলেন যে গত দুই দিনে যা লিখেছেন তা হয়তো সঠিক নয়। এখন আপনি যদি সেসব অংশ মুছে ফেলেন, তাহলে আপনি হয়তো পূর্ববর্তী লেখার গুরুত্বপূর্ণ অংশও মুছে ফেলতে পারেন।

এখন, এই পরিস্থিতিতে আপনি কী করবেন? যদি আপনি মনে না রাখেন যে ঠিক কোন অংশটি মুছা উচিত, অথবা দুই দিন আগে আপনার লেখা ঠিক কী ছিল, তাহলে সমস্যাটি কতটা জটিল হবে।

এখানেই গিট সাহায্য করে। গিট প্রতিটি ফাইলের পরিবর্তনগুলো মনিটর করে এবং সেগুলোকে ভার্সনাইজ করে রাখে। এর ফলে, আপনি যেকোনো সময় পূর্ববর্তী অবস্থায় ফিরে যেতে পারেন, এবং আপনার কোনো গুরুত্বপূর্ণ তথ্য হারানোর সম্ভাবনা নেই।

---

## 1. **গিট ইনিশিয়ালাইজ করা**

```bash
 git init
```


**ব্যাখ্যা:**
প্রথমে আপনি যখন গিট প্রজেক্টে কাজ শুরু করবেন, আপনাকে প্রজেক্টটি গিট দিয়ে ইনিশিয়ালাইজ করতে হবে। এটি কমান্ড দিলে আপনার প্রজেক্ট ফোল্ডারে একটি `.git` ডিরেক্টরি তৈরি হবে, যা গিটের রেপোজিটরি হিসেবে কাজ করবে।

**সেরা অভ্যাস:**
প্রোজেক্টের শুরুর সময়েই `git init` করে নিতে হবে যাতে সকল ফাইলের পরিবর্তন সঠিকভাবে ট্র্যাক হয়। প্রকল্পের মূল ফোল্ডারে কমান্ডটি চালানো উচিত।

---

## 2. **গিট এর স্ট্যাটাস চেক করা**

```bash
git status
```

**ব্যাখ্যা:**
এটি আপনাকে জানাবে কোন ফাইলগুলি মডিফাই হয়েছে, কোন ফাইল নতুন যোগ হয়েছে, এবং কোন ফাইল এখনও কমিট করা হয়নি। এটি গিটের বর্তমান স্টেজ এবং রিপোজিটরির অবস্থা দেখার জন্য একটি দরকারী কমান্ড।

**সেরা অভ্যাস:**
কোনো নতুন পরিবর্তন যোগ করার পরে বা কোড এডিট করার পর `git status` চালিয়ে ফাইলগুলোর অবস্থা চেক করুন যাতে ভুল করে কিছু বাদ না পরে যায়।

---

## 3. **ফাইলকে গিটের ট্র্যাক করার জন্য স্টেজ এ উঠানো**

```bash
git add <filename> // নির্দিষ্ট ফাইল স্টেজে আনা
git add --all      // সব ফাইল স্টেজে আনা
git add .          // বর্তমান ডিরেক্টরির সব ফাইল স্টেজে আনা
```

**ব্যাখ্যা:**
`git add` কমান্ডটি ব্যবহার করে আপনি ফাইল বা ফোল্ডারগুলোকে স্টেজে নিয়ে আসতে পারবেন, যা পরবর্তীতে কমিটের জন্য প্রস্তুত করা হয়।

**সেরা অভ্যাস:**
ফাইল যোগ করার আগে নিশ্চিত হন যে, শুধুমাত্র প্রয়োজনীয় ফাইলগুলোই স্টেজে উঠছে এবং ভুল ফাইলগুলি বাদ পড়ে না।

---

## 4. **কমিট করা**

```bash
git commit -m "<commit message>"
```

**ব্যাখ্যা:**
এই কমান্ডটি দিয়ে আপনার করা পরিবর্তনগুলো গিটে স্থায়ীভাবে সংরক্ষণ করা হয়। `-m` এর মাধ্যমে আপনি একটি ছোট্ট বার্তা দিতে পারবেন, যা সেই কমিটের বিবরণ দেয়।

**সেরা অভ্যাস:**
প্রতিটি কমিটের জন্য সংক্ষিপ্ত এবং বোধগম্য কমিট বার্তা দিন, যাতে ভবিষ্যতে কমিটের ইতিহাস দেখে বোঝা যায় যে কোন কমিটে কি পরিবর্তন করা হয়েছে।

---

## 5. **গিট লগ চেক করা**

```bash
git log
git log --oneline
```

**ব্যাখ্যা:**
`git log` কমান্ডটি দিয়ে আপনি পূর্বের সমস্ত কমিটের সম্পূর্ণ বিবরণ দেখতে পারবেন। আর `--oneline` ফ্ল্যাগ ব্যবহার করলে প্রতিটি কমিটের সংক্ষিপ্ত তথ্য এক লাইনে দেখা যায়।

---

## 6. **একটা কমিট থেকে অন্য কমিটের অবস্থায় ফিরে যাওয়া**

```bash
git reset --hard <commit id>
```

**ব্যাখ্যা:**
এই কমান্ড দিয়ে আপনি নির্দিষ্ট একটি কমিটে ফিরে যেতে পারবেন এবং বর্তমান কাজের সব পরিবর্তন মুছে ফেলবেন। এটি একটি শক্তিশালী কমান্ড, তাই ব্যবহারের আগে সতর্কতা অবলম্বন করুন।

**সেরা অভ্যাস:**
এই কমান্ড চালানোর আগে ভালো করে যাচাই করে নিন, কারণ `--hard` ফ্ল্যাগ ব্যবহার করলে মুছে ফেলা ফাইলগুলো সহজে পুনরুদ্ধার করা সম্ভব হবে না।

---

## 7. **গিটের প্রতিটা চেঞ্জের রেফারেন্স সহ লগ দেখার জন্য**

```bash
git reflog
```

**ব্যাখ্যা:**
`git reflog` দিয়ে আপনি প্রতিটি চেঞ্জের রেফারেন্স দেখতে পারবেন। এটি প্রতিটি কমান্ডের ফলাফল দেখায়, যেমন কোন ব্রাঞ্চে কাজ হয়েছে, এবং কখন সুইচ করা হয়েছে।

---

## 8. **গিটের ট্র্যাক থেকে কোন ফাইল রিমুভ করা**

```bash
git rm <filename>
```

**ব্যাখ্যা:**
`git rm` দিয়ে আপনি গিটের ট্র্যাক থেকে কোনো ফাইল সরিয়ে দিতে পারবেন। এটি ফাইলকে রিপোজিটরি থেকে পুরোপুরি মুছে ফেলে।

**সেরা অভ্যাস:**
ফাইল ডিলিট করার আগে স্টেজে থাকা ফাইলের অবস্থা যাচাই করুন। প্রয়োজন হলে `git add` করে অন্যান্য পরিবর্তন কমিট করে তারপর `git rm` ব্যবহার করুন।

---

## 9. **ব্রাঞ্চ তৈরী করা**

```bash
git branch <branch name>
```

**ব্যাখ্যা:**
এই কমান্ডটি দিয়ে আপনি একটি নতুন ব্রাঞ্চ তৈরি করতে পারবেন, যেখানে আপনি স্বাধীনভাবে কাজ করতে পারবেন। উদাহরণস্বরূপ, `bugfix/fixname` নামে একটি ব্রাঞ্চ তৈরি করা যেতে পারে।

---

## 10. **ব্রাঞ্চ লিস্ট দেখা**

```bash
git branch --list
```

**ব্যাখ্যা:**
`git branch --list` দিয়ে আপনি গিটের সমস্ত ব্রাঞ্চের তালিকা দেখতে পারবেন।

---

## 11. **এক ব্রাঞ্চ থেকে অন্য ব্রাঞ্চে সুইচ করা**

```bash
git switch <branch name>
```

**ব্যাখ্যা:**
এই কমান্ডটি ব্যবহার করে আপনি একটি ব্রাঞ্চ থেকে অন্য ব্রাঞ্চে পরিবর্তন করতে পারবেন। এটি আগের `git checkout` এর বিকল্প।

<Callout type="info">
  💡 সুইচ করার আগে নিশ্চিত হন যে, আপনার বর্তমান কাজ কমিট করা হয়েছে, যাতে কোনো
  পরিবর্তন হারিয়ে না যায়।
</Callout>

---

## 12. **এক ব্রাঞ্চের কোড অন্য ব্রাঞ্চে মার্জ করা**

```bash
git switch <target branch>  # যেই ব্রাঞ্চে কোড মার্জ করতে চান সেখানে যান
git merge <source branch>   # যেই ব্রাঞ্চের কোড মার্জ করতে চান সেটি উল্লেখ করুন
```

**ব্যাখ্যা:**
মার্জ করার মাধ্যমে এক ব্রাঞ্চের পরিবর্তনগুলো অন্য ব্রাঞ্চে একত্রিত করা হয়।

---

## 13. **গিট থেকে ব্রাঞ্চ ডিলিট করা**

```bash
git branch -d <branch name>
```

**ব্যাখ্যা:**
`git branch -d` কমান্ডটি দিয়ে আপনি গিট থেকে নির্দিষ্ট ব্রাঞ্চ মুছে ফেলতে পারবেন।

---

## 14. **ব্রাঞ্চ নেম পরিবর্তন করতে চাইলে**

```bash
git branch -m <updated branch name>
```

**ব্যাখ্যা:**
এই কমান্ড দিয়ে আপনি ব্রাঞ্চের নাম পরিবর্তন করতে পারবেন। প্রথমে সেই ব্রাঞ্চে যেতে হবে, তারপর `git branch -m` কমান্ড চালাতে হবে।

---

## 15. **গিট স্ট্যাশ করা (আনকমিটেড কাজ গিটের ড্রাফট এ সেভ রাখা)**

```bash
git stash
```

**ব্যাখ্যা:**
এই কমান্ডটি দিয়ে আপনার কাজের অগ্রগতি স্ট্যাশ করে রাখা যায়, যাতে আপনি পরবর্তীতে তা পুনরুদ্ধার করতে পারেন।

---

## 16. **গিট স্ট্যাশ লিস্ট চেক করা**

```bash
git stash list
```

**ব্যাখ্যা:**
`git stash list` দিয়ে আপনি গিটের সমস্ত স্ট্যাশের তালিকা দেখতে পারেন।

---

## 17. **গিটের স্ট্যাশে কি কি সেভ রাখা হয়েছিল তা দেখতে**

```bash
git stash show -p
```

## 18. **গিটের স্ট্যাশ এপ্লাই করার জন্য**

```bash
git stash apply <stash id>
or
git stash pop
```

## 21. **রিমোট রিপোজিটরি থেকে পুল করা**

```bash
git pull
```

`git pull` কমান্ডটি ব্যবহার করে আপনি রিমোট রিপোজিটরি থেকে সর্বশেষ পরিবর্তনগুলো আপনার লোকাল মেশিনে নিয়ে আসতে পারেন।
